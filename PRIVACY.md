# Privacy Policy — REDCap Browser Extension

**Effective date:** 2026-07-08

## Summary

The REDCap Browser Extension does not collect, transmit, or sell any user data to its developer or to any third party. All data the extension handles stays on your device and, when transmitted, goes only to the REDCap server *you* configure — never to the extension developer.

## What data the extension stores

The extension saves the following in your browser's `chrome.storage.sync` (synced by your browser account, e.g. your Google or Firefox account — not by the extension developer):

- **Configuration key(s)**: the REDCap server URL, project ID, and an extension-specific access token you obtain from your own REDCap server's "Browser Extension Configuration" page.
- **Profile names**: labels you assign to each configured server (e.g. "Production", "Development") if you use the multi-profile feature.
- **A display preference**: whether links open in a new tab or the current tab.

None of this data is collected by, or ever sent to, the developer of this extension. It is stored locally/synced via your browser's own sync feature, under your control, and can be cleared at any time by removing the extension or clearing its storage.

## What data the extension transmits, and to whom

The extension sends HTTPS POST requests directly from your browser to the REDCap server you configured (the `baseUrl` in your configuration key), using the extension-specific access token, to:

- Look up and autocomplete project names you have access to.
- Retrieve permission/role information (e.g. whether you are a system administrator or project designer) needed to show or hide relevant shortcuts.
- Create new records when you use the "New Record" shortcut.

These requests go **only** to the REDCap server you yourself configured — an institution's own server that you control or that your organization operates. They never pass through, or get logged by, any server operated by the extension developer. The extension developer has no access to your REDCap data, records, or credentials.

As described in the extension's README, the access token used is a dedicated, extension-specific token generated by the REDCap Browser Extension Support external module. It only grants project-listing and navigation actions — it cannot export or import study data — and can be revoked independently by your REDCap administrator at any time.

## Scripting permission

The extension requests the `scripting` permission and broad host permissions (`*://*/*`) for one purpose only: to check whether the current tab is displaying your REDCap server's "Browser Extension Configuration" page, so it can offer one-click automatic setup by reading a hidden configuration element on that page. This check runs only when you open the extension's popup, reads nothing else on the page, and sends nothing anywhere as a result of the check itself — it merely populates fields in the extension's own local configuration UI, which you then choose to save.

The extension does not monitor your browsing activity, read page content on other sites, or track which sites you visit.

## Third parties

The extension shares no data with any third party. It does not use analytics, advertising, or tracking services of any kind.

## Limited Use compliance

This extension's use and transfer of information received from Google APIs (and browser APIs generally) adheres to the [Chrome Web Store User Data Policy](https://developer.chrome.com/docs/webstore/program-policies/), including the Limited Use requirements: data is used solely to provide and improve the extension's disclosed navigation feature, is never sold, and is never used for advertising or unrelated purposes.

## Changes to this policy

If this policy changes, the "Effective date" above will be updated and the revised policy will be published at this same location.
